# SwarmAI MVP Use Cases (Operator-Focused)
*For PMs, Marketers, Sales, and Managers — Non-Technical Knowledge Work*

---

## 1. MVP Goal

Validate the core promise:

> “SwarmAI acts as your AI execution team that continuously triages, plans, executes, and follows through on daily work inside a persistent workspace.”

The MVP should prove three capabilities:
1. Capture real work signals (emails, meetings, chat, manual ToDos)
2. Convert signals → executable tasks with AI clarification
3. Produce tangible outputs (reports, plans, follow-ups) with minimal supervision

---

# 2. Core MVP Design Scope

The MVP centers on three core surfaces:

| Surface | Purpose |
|--------|---------|
| **SwarmWS (Left)** | Persistent workspace memory & project context |
| **Chat Command Center (Middle)** | Explore → clarify → execute workflows |
| **Swarm Radar (Right)** | Glanceable triage: ToDos, WIP tasks, waiting input, completed |

---

# 3. MVP Use Case 1 — Daily Work Triage Command Center
## (All Personas: PM / Marketing / Sales / Manager)

### Scenario
User opens SwarmAI in the morning to see:
- Follow-ups from email
- Meeting action items
- Chat-created ToDos
- Ongoing execution tasks

### Flow
1. External signals → converted into ToDos in Swarm Radar
2. User scans “Needs Attention”
3. Click “Start” on a ToDo
4. Chat session opens with context preloaded
5. AI clarifies requirements if ambiguous
6. Converts to WIP Task and begins execution

### Example
> “Prepare weekly team status report”  
AI gathers: last week’s updates, meeting notes, pending tasks → drafts report.

### Validation Signal
User feels:  
> “This replaced my morning triage across Slack, email, and docs.”

# 4. MVP Use Case 2 — PM Weekly Review & Planning
## (Primary Persona: Product Managers)

### Scenario
PM needs to prepare weekly review: progress, risks, next steps.

### Flow
1. PM clicks ToDo: “Prepare weekly product review”
2. Chat session opens → AI asks clarification:
   - Audience?
   - Scope (all projects or one)?
3. AI gathers:
   - Project context (SwarmWS + Project folder)
   - Completed tasks (last 7 days)
   - Open risks / blockers
4. AI drafts structured review report
5. PM edits → final artifact saved in project

### Output Artifact
- Weekly product review document
- Key risks + next actions list

### Validation Signal
PM saves 1–2 hours weekly on synthesis & reporting.

# 5. MVP Use Case 3 — Marketing Campaign Orchestration
## (Primary Persona: Marketing Managers)

### Scenario
Marketer wants to launch a campaign and track ongoing work.

### Flow
1. User types:
   > “Launch Q2 product marketing campaign”
2. AI creates:
   - Task plan (content, channels, timeline)
   - ToDos auto-generated (content drafts, stakeholder review, launch checklist)
3. Each ToDo becomes executable tasks
4. AI drafts:
   - Campaign brief
   - Content outline
   - Weekly campaign performance summary

### Output Artifacts
- Campaign plan document
- Content calendar
- Weekly performance report

### Validation Signal
User experiences SwarmAI as:
> “My AI marketing operations team.”

# 8. MVP Use Case 6 — Ad-hoc Exploration → Execution Conversion
